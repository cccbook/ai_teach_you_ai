# 12. 預訓練的力量

## 從 microgpt 到真正的 LLM

microgpt 展示了 GPT 的核心原理，但規模差距巨大：

| 特性 | microgpt | GPT-3 |
|-----|---------|-------|
| 參數量 | 4,192 | 175,000,000,000 |
| 訓練資料 | 32,000 個名字 | 數兆 tokens |
| 詞彙表 | 27 | ~50,000 |
| 層數 | 1 | 96 |
| 維度 | 16 | 12,288 |

預訓練就是用**海量資料**訓練這些超大模型。

## 預訓練的目標：下一個 Token 預測

GPT 系列的預訓練目標非常簡單：**給定前面的文字，預測下一個 token。**

```
輸入：今天天氣很好，我決定去
輸出：公園
```

這個任務看似簡單，卻讓模型學會了：
- 文法結構
- 世界知識
- 推理能力
- 邏輯思維

因為要正確預測下一個詞，模型必須理解上下文、語義、甚至世界運作的方式。

## 無監督學習：不需要標籤

預訓練是**無監督學習**——不需要人工標註答案。

訓練資料就是網路上的所有文字：
- 網頁
- 書籍
- 論文
- 程式碼
- 對話記錄

資料本身就是答案：下一個 token 就在原文裡。

## 規模法則：越大越好

OpenAI 發現的規模法則：

```
模型越大 + 資料越多 + 計算量越大 = 模型越強
```

| 發現 | 說明 |
|-----|------|
| 損失與參數量 | 損失大致服從冪律分佈 |
|湧現能力 | 當規模超過某個閾值時，新能力突然出現 |
| 資料品質 | 高品質資料比低品質資料更有效 |

### 湧現能力示例

| 能力 | 需要的參數量 |
|-----|------------|
| 基礎語法 | 任何規模 |
| 基本推理 | ~10B |
| 複雜推理 | ~100B |
| 程式設計 | ~10B+ |
| 數學證明 | ~100B+ |

## Scaling Laws

Kaplan 等人提出的規模法則：

$$L(N) \approx \left(\frac{N_c}{N}\right)^{\alpha_N}$$

其中 $L$ 是損失，$N$ 是參數量，$\alpha_N \approx 0.076$。

簡單說：**模型參數量每增加 10 倍，損失降低約一半。**

## GPT-2 到 GPT-4

### GPT-2 (2019)
- 15 億參數
- 訓練資料：800 萬網頁
- 能力：基本文字生成

### GPT-3 (2020)
- 1750 億參數
- 訓練資料：3000 億 tokens
- 能力：Few-shot 學習

### GPT-4 (2023)
- 估計 1-2 兆參數
- 多模態輸入
- 能力：複雜推理、程式設計、數學

## 預訓練的代價

預訓練需要龐大的計算資源：

| 模型 | GPU 小時 | 碳排放（估計）|
|-----|---------|--------------|
| GPT-3 | ~3.64E+07 | ~500 噸 CO2 |
| PaLM | ~2.85E+07 | ~300 噸 CO2 |

這也是為什麼只有少數公司能訓練真正的 LLM。

## 預訓練後的模型：Base 模型

預訓練完成的模型叫做 **Base 模型** 或 **Foundation 模型**。

特性：
- 強大的文字生成能力
- 但回答不夠友善
- 可能會產生有害內容
- 不擅長遵循指令

```python
# Base 模型輸入
輸入："如何撬鎖？"

# Base 模型輸出（不安全）
輸出："撬鎖的方法如下：..."
```

所以預訓練只是第一步，還需要**微調**讓模型更安全、更有用。

## 總結

這一章的核心概念：
- **預訓練目標**：下一個 token 預測
- **無監督學習**：不需要人工標籤
- **規模法則**：越大越好
- **湧現能力**：規模超過閾值時能力突然出現
- **Base 模型**：預訓練後的原始模型
