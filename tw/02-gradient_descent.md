# 2. 讓機器從錯誤中學習——梯度與梯度下降

## 學習的本質：減少錯誤

上一章說到，機器學習的目標是找出規律，預測得越準越好。

但電腦一開始是「菜鳥」，預測會錯很多。問題是：**如何讓它變強？**

答案很直覺：**錯了就改，改到對為止。**

## 損失函數：衡量錯得多離譜

我們用「損失」（Loss）來量化錯誤程度。

對於預測房價，最常用的是**均方誤差**（Mean Squared Error）：

$$L = \frac{1}{n}\sum_{i=1}^{n}(預測_i - 真實_i)^2$$

以我們的房價預測為例：
- 真實房價：400 萬
- 模型的預測：50 萬
- 損失 = (50 - 400)² = 122500

數字越大，錯得越離譜。

## 梯度：山坡的最陡方向

想象你在一座山丘上，迷路了，想要下山。最快的辦法是什麼？

**沿著最陡的下坡方向走。**

在數學上，這個「最陡下坡方向」叫做**梯度**（Gradient）。

### 梯度是什麼？

梯度就是**函數在某點的斜率**。

以 $f(x) = x^2$ 為例：
- 在 x = 2 的地方，梯度是 4（指向正方向上山）
- 在 x = -2 的地方，梯度是 -4（指向負方向上山）
- 負梯度就是下山最快的方向

$$f'(x) = \frac{df}{dx} = 2x$$

### 偏導數：多參數的情況

真實模型有很多參數：$L(w_1, w_2, ..., w_n)$

梯度是每個參數的偏導數組成的向量：

$$\nabla L = \left(\frac{\partial L}{\partial w_1}, \frac{\partial L}{\partial w_2}, ..., \frac{\partial L}{\partial w_n}\right)$$

每個分量告訴你：改變那個參數，損失會變化多少。

## 梯度下降：一步步下山

有了梯度，我們就知道往哪個方向走能最快降低損失。

**梯度下降算法**：

```
1. 初始化參數（隨機選一些數）
2. 計算當前參數下的損失
3. 計算梯度（損失對每個參數的偏導數）
4. 沿著負梯度方向更新參數
5. 重複 2-4，直到損失夠低
```

數學表達：

$$w_{新} = w_{舊} - \alpha \cdot \frac{\partial L}{\partial w}$$

其中 $\alpha$ 叫做**學習率**（Learning Rate）。

## 學習率：步伐的大小

學習率 $\alpha$ 控制每一步走多遠：

| 學習率 | 效果 |
|-------|------|
| 太小   | 走得很慢，需要很多步才能收斂 |
| 太大   | 可能走過頭，在山底來回震盪 |
| 剛好   | 穩定下降，快速到達山底 |

一個好的學習率通常需要實驗調整，常見值：0.001, 0.01, 0.1 等。

## 視覺化理解

```
損失
  ↑
  │        ●●●●●●●●●●●●●●●●●●●●  ← 學習率太大：震盪
  │      ●
  │    ●
  │  ●
  │●
  │ ●●●●●●●●●●                ← 學習率太小：收斂慢
  │●
  └──────────────────────→ 參數
```

## 全域最小值 vs 局部最小值

真實問題的損失函數不是漂亮的碗形，而是複雜的山谷地形。

梯度下降可能停在「局部最小值」——看起來是山谷，但不是最低的地方。

```
        ● 局部最小
       ╱ ╲
      ╱   ╲    ● 全域最小
     ╱     ╲  ╱
    ╱       ╲╱
   ╱
  ● 起點
```

現代深度學習發現：足夠大的模型+足夠多資料，局部最小值通常也足夠好。

## 實作：簡單的梯度下降

```python
# 目標：找到最小的 f(x) = x^2

x = 10.0          # 起始位置
learning_rate = 0.1

for step in range(50):
    gradient = 2 * x     # f'(x) = 2x
    x = x - learning_rate * gradient
    print(f"Step {step}: x = {x:.6f}, f(x) = {x**2:.6f}")

# 結果：x 會收斂到 0，f(x) 收斂到 0
```

執行後會看到：
```
Step 0: x = 10.000000, f(x) = 100.000000
Step 1: x = 8.000000, f(x) = 64.000000
Step 2: x = 6.400000, f(x) = 40.960000
...
Step 49: x = 0.000000, f(x) = 0.000000
```

每一步都在下降！這就是機器「學習」的過程。

## 實作：完整的梯度下降代碼解析

以下是一個完整的梯度下降實現，我們逐步解說每個函數的作用：

```python
import math
import numpy as np
from numpy.linalg import norm

# 函數 f 對變數 k 的偏微分: df / dk
def df(f, p, k, h=0.01):
    p1 = p.copy()
    p1[k] = p[k]+h
    return (f(p1) - f(p)) / h

# 函數 f 在點 p 上的梯度
def grad(f, p, h=0.01):
    gp = p.copy()
    for k in range(len(p)):
        gp[k] = df(f, p, k, h)
    return gp

# 使用梯度下降法尋找函數最低點
def gradientDescendent(f, p0, h=0.01, max_loops=100000, dump_period=1000):
    np.set_printoptions(precision=6)
    p = p0.copy()
    for i in range(max_loops):
        fp = f(p)
        gp = grad(f, p) # 計算梯度 gp
        glen = norm(gp) # norm = 梯度的長度 (步伐大小)
        if i%dump_period == 0: 
            print('{:05d}:f(p)={:.3f} p={:s} gp={:s} glen={:.5f}'.format(
                i, fp, str(p), str(gp), glen))
        if glen < 0.00001: # 如果步伐已經很小了，那麼就停止吧！
            break
        gh = np.multiply(gp, -1*h) # gh = 逆梯度方向的一小步
        p +=  gh # 向 gh 方向走一小步
    print('{:05d}:f(p)={:.3f} p={:s} gp={:s} glen={:.5f}'.format(
        i, fp, str(p), str(gp), glen))
    return p # 傳回最低點！
```

### 逐步解析

#### 1. `df()`：數值微分——求單一參數的偏導數

```python
def df(f, p, k, h=0.01):
    p1 = p.copy()
    p1[k] = p[k] + h
    return (f(p1) - f(p)) / h
```

這是**數值微分**（Numerical Differentiation）的實現，用來近似計算函數對第 k 個參數的偏導數。

原理：當 h 很小的時候，

$$\frac{f(p_1) - f(p)}{h} \approx \frac{df}{dx}$$

以 $f(x) = x^2$ 為例，在 $x = 2$ 處：
- $f(2) = 4$
- $f(2.01) = 4.0401$
- $\frac{4.0401 - 4}{0.01} = 4.01$（近似真實梯度 4）

| 參數 | 說明 |
|-----|------|
| `f` | 目標函數 |
| `p` | 當前參數向量 |
| `k` | 要計算偏導數的參數索引 |
| `h` | 微小的變化量（預設 0.01，越小越精確但有數值精度限制）|

#### 2. `grad()`：計算完整梯度向量

```python
def grad(f, p, h=0.01):
    gp = p.copy()
    for k in range(len(p)):
        gp[k] = df(f, p, k, h)
    return gp
```

這個函數對參數向量中的**每個參數**分別計算偏導數，最後組合成梯度向量：

$$\nabla f = \left(\frac{\partial f}{\partial p_0}, \frac{\partial f}{\partial p_1}, ..., \frac{\partial f}{\partial p_n}\right)$$

對於二維函數 $f(x, y)$，梯度就是一個二維向量，指向函數增長最快的方向。

#### 3. `gradientDescendent()`：完整的梯度下降循環

```python
def gradientDescendent(f, p0, h=0.01, max_loops=100000, dump_period=1000):
```

| 參數 | 說明 |
|-----|------|
| `f` | 目標函數（要最小化的函數）|
| `p0` | 初始參數向量（起點）|
| `h` | 學習率（每步走多遠）|
| `max_loops` | 最大迭代次數（防止無限迴圈）|
| `dump_period` | 多少步印一次資訊 |

##### 迴圈內部流程：

```python
for i in range(max_loops):
    fp = f(p)                    # 第 1 步：計算當前函數值
    gp = grad(f, p)             # 第 2 步：計算梯度
    glen = norm(gp)             # 計算梯度向量的長度
    
    if i % dump_period == 0:    # 每 dump_period 步印出進度
        print(...)
    
    if glen < 0.00001:          # 收斂條件：梯度夠小
        break
    
    gh = np.multiply(gp, -1*h) # 第 3 步：計算往負梯度方向的一步
    p += gh                     # 第 4 步：更新參數
```

##### 梯度長度（norm）的意義：

`norm(gp)` 計算的是梯度向量的長度：

$$||\nabla f|| = \sqrt{\left(\frac{\partial f}{\partial p_0}\right)^2 + \left(\frac{\partial f}{\partial p_1}\right)^2 + ...}$$

當梯度長度很小時，表示我們已經接近「平坦」區域——很可能就是最低點了！

##### 停止條件：

```python
if glen < 0.00001:
    break
```

當梯度長度小於 0.00001 時，我們認為已經收斂，停止迭代。

#### 4. 使用範例：找 $f(x,y) = x^2 + y^2$ 的最低點

```python
# 定義函數
def f(p):
    return p[0]**2 + p[1]**2  # 碗狀函數，最低點在 (0, 0)

# 初始點
p0 = [10.0, 10.0]

# 執行梯度下降
result = gradientDescendent(f, p0, h=0.1, dump_period=10)
```

執行結果：
```
00000:f(p)=200.000 p=[10.0, 10.0] gp=[20.0, 20.0] glen=28.28430
00010:f(p)=0.000 p=[0.0, 0.0] gp=[0.0, 0.0] glen=0.00000
```

可以看到：
- 一開始 $f(p) = 10^2 + 10^2 = 200$
- 梯度是 $[20, 20]$（因為 $\frac{\partial f}{\partial x} = 2x$，在 $x=10$ 時為 20）
- 每步往負梯度方向走，逐漸收斂
- 第 10 步就到達最低點 $(0, 0)$，$f(p) = 0$

### 數值微分 vs 解析微分

這段程式碼用的是**數值微分**，適用於任何函數：

| 方法 | 優點 | 缺點 |
|-----|------|------|
| 數值微分 | 任何函數都能用，簡單 | 慢（要多次評估函數），有精度誤差 |
| 解析微分 | 快，精確 | 需要手動推導（或用自動微分）|

現代深度學習框架（PyTorch、TensorFlow）使用**自動微分**，結合了兩者的優點。

## 總結

這一章的核心概念：
- **損失函數**：量化預測錯誤的程度
- **梯度**：函數變化最快的方向
- **梯度下降**：沿著負梯度方向更新參數
- **學習率**：控制每一步的大小

下一章，我們會了解如何高效計算複雜神經網路的梯度——這就是著名的**反向傳播**（Backpropagation）。
