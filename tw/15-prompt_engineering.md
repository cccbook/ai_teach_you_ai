# 15. Prompt Engineering：與模型對話的藝術

## 為什麼 Prompt 很重要？

同樣的模型，不同的 Prompt 效果可能天差地別：

```python
# Prompt 1：效果差
回答：「量子計算」
```

```python
# Prompt 2：效果好
"""
解釋量子計算的重點：
1. 用普通人能理解的語言
2. 提供具體例子
3. 不要超過 200 字

問題：什麼是量子計算？
"""
回答："量子計算就像用很多硬幣同時硬幣..."
```

Prompt Engineering 就是**學會如何有效地與 LLM 溝通**。

## 基本原則

### 1. 明確具體

| ❌ 模糊 | ✅ 具體 |
|--------|--------|
| "寫一首詩" | "寫一首五言絕句，主題是春天的第一場雨" |
| "幫我寫 code" | "用 Python 寫一個計算斐波那契數列的函數，包含 type hint" |
| "摘要這篇文章" | "用三句話摘要這篇文章，重點是作者的論點" |

### 2. 提供上下文

```python
# 沒有上下文
"翻譯：Hello"

# 有上下文
"""
你是一個專業的中英翻譯員，擅長科技文章。
請將以下英文翻譯成中文，注意術語一致性。

原文：The transformer architecture revolutionized NLP.
譯文：
"""
```

### 3. 指定輸出格式

```python
# 沒有格式要求
"列出 5 個減肥方法"

# 有格式要求
"""
列出 5 個減肥方法，格式如下：
{
  "method_1": "方法名稱",
  "description": "具體說明"
}
"""
```

## 常用技巧

### Few-Shot Prompting

提供幾個範例，讓模型學習模式：

```python
prompt = """
把以下中文翻譯成英文：

範例 1:
中文：「今天天氣很好」
英文：The weather is nice today.

範例 2:
中文：「我愛學習新技能」
英文：I love learning new skills.

現在請翻譯：
中文：「這本書非常有趣」
英文：
"""
```

### Chain-of-Thought (CoT)

讓模型一步步思考：

```python
prompt = """
問題：小明有 10 個蘋果，給了小華 3 個，又買了 5 個，現在有多少蘋果？

讓我們一步步思考：
1. 小明一開始有 10 個蘋果
2. 給了小華 3 個，所以剩下 10 - 3 = 7 個
3. 又買了 5 個，所以變成 7 + 5 = 12 個

答案：12 個

問題：一輛車每小時行駛 60 公里，行駛 2.5 小時後走了多遠？
讓我們一步步思考：
"""
```

### Zero-Shot CoT

即使沒有範例，加上「讓我們一步步思考」也能提升效果：

```python
prompt = """
問題：三個人住飯店，每人付 10 元，共 30 元。老闆說只要 25 元，退回 5 元。
服務生偷了 2 元，找回每人 1 元。所以每人實際付 9 元，3*9=27 元，
服務生偷了 2 元，27+2=29 元。問：剩下的 1 元去哪了？

讓我們一步步思考，不要急於給答案：
"""
```

### Tree of Thoughts (ToT)

探索多條思考路徑：

```python
prompt = """
解決這個問題時，請考慮多個可能的方向：

問題：如何讓公司營收增長 50%？

方向 1：開源
- 增加產品線
- 提高售價
- 拓展新市場

方向 2：節流
- 降低生產成本
- 優化人力配置
- 減少浪費

方向 3：創新
- 開發新技術
- 改善客戶體驗
- 建立合作夥伴關係

請針對每個方向詳細說明執行步驟。
"""
```

## 避免常見錯誤

### 1. 上下文長度限制

```python
# ❌ 太長，超出模型上下文限制
prompt = "以下是 100 頁的合同內容..." + "請總結要點"

# ✅ 明確指出重點
prompt = """
以下是合同的關鍵條款摘要（第 1-3 頁）：
[摘要內容]

請根據以上摘要回答：這個合同的主要義務是什麼？
"""
```

### 2. 模糊的約束

| ❌ 模糊約束 | ✅ 具體約束 |
|-----------|-----------|
| "不要太長" | "回答限制在 100 字以內" |
| "專業一點" | "使用正式商業語言，避免口語化表達" |
| "快點" | （無法控制，模型有自己的速度）|

### 3. 衝突的指令

```python
# ❌ 衝突
prompt = "詳細解釋這個概念" + "用一句話說明"

# ✅ 協調
prompt = "用一句話說明核心概念，如果需要的話可以補充一個簡短例子"
```

## 高級技巧

### System Prompt

為 AI 設定角色和行為：

```python
messages = [
    {"role": "system", "content": "你是一個資深軟體工程師，擅長 Python 和系統設計。回答時會提供程式碼範例。"},
    {"role": "user", "content": "如何設計一個高併發系統？"}
]
```

### Refinement Loop

逐步改進回答：

```python
# 第一輪：給出初步回答
回答1 = model.generate("問題：...")

# 根據反饋改進
回答2 = model.generate(f"""
原始回答：{回答1}

請根據以下反饋改進回答：
- 補充更多具體例子
- 簡化技術術語
""")
```

## 總結

這一章的核心概念：
- **明確具體**：清楚告訴模型你要什麼
- **提供上下文**：讓模型理解背景
- **Few-Shot**：用範例引導輸出
- **Chain-of-Thought**：鼓勵一步步思考
- **System Prompt**：設定角色和行為
